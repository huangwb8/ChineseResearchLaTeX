# Prompt 模板与最佳实践

> 目标：让 `get-review-theme` 从任意输入源稳定输出 `主题 + 关键词 + 核心问题`。本文件只保留最小可执行模板与选词原则。

## 通用 Prompt

```text
请分析以下内容，提取结构化综述主题。

【输入内容】
{content}

【输出要求】
主题：{一句话概括，包含研究对象 + 核心问题/方法}
关键词：{5-10 个英文标准术语}
核心问题：{2-5 个具体挑战或科学问题}

【质量要求】
- 主题不要过宽
- 关键词优先标准学术术语
- 核心问题必须具体、可检索、可讨论
```

## 按输入类型微调

### 自然语言描述

- 优先识别：研究对象、核心方法、应用场景、隐含挑战

### PDF / Word

- 优先读：标题、摘要、结论
- 术语尽量沿用原文标准表达

### 图片

- 先描述图中对象、模块与关系，再抽主题

### URL / 文件夹

- 先过滤无关内容，再抽共同主题与核心问题

## 主题生成原则

- 最好写成“研究对象 + 方法/问题”
- 避免只剩大词，例如“深度学习”“医学影像”“自然语言处理”

示例：

- 不好：`深度学习`
- 较好：`深度学习在医学影像中的应用`
- 更好：`深度学习在医学影像小样本癌症诊断中的应用`

## 关键词原则

- 统一用英文标准术语
- 优先可用于 PubMed / Web of Science / ACM 检索
- 避免口语词和非标准缩写

## 核心问题原则

- 必须是具体挑战，不是“如何更好”“提升性能”
- 可以从样本、泛化、解释性、跨平台、因果、低资源等方向抽取

## 降级提示

如果输入过短、过乱或主题过宽，可提示用户补：

- 研究对象
- 核心方法
- 应用场景
- 最关心的挑战
