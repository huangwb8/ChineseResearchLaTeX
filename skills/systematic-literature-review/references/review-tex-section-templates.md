# `{主题}_review.tex` Section 模板（Premium 档位）

> 目的：给 `systematic-literature-review` 的 `{主题}_review.tex` 提供**逐 section、可直接套用**的高质量模板，减少"写作结构不稳/像笔记"的概率，从而让 skill 的产出更接近顶刊 Review 的可读性与可审稿性。
>
> 说明：本模板以 `review_example_01.pdf` 的结构信号为锚，不复刻其原文内容，只抽取"结构与写作工程信号"（稳定层级、主题-子主题组织、批判性综合、Questions/Perspectives 收束等），并转写成通用的可复用框架。

---

## 1. 你要模仿的“结构信号”是什么？

`review_example_01.pdf`（Cell Review）呈现的关键结构信号（可被审稿人快速感知）：

- **开头清晰**：先用 `Summary` 给出中心主张与争议点/新主题，再用 `Introduction` 定义“旧共识 → 新数据 → 争议 → 本综述要解决的问题”。
- **主体是 taxonomy 驱动**：不是“结果堆砌”，而是用 4–7 个一级主题（\section）把证据组织成可检验的分类框架；每个主题下再用 2–4 个子主题（\subsection）展开。
- **批判性收束**：专门的 `Questions and concerns`（或同等功能段）集中讨论证据的局限、方法学陷阱、争议来源与可复现性风险。
- **前瞻性落点**：`Perspectives`（或同等功能段）给出可检验的研究议程与可落地的实践含义。

---

## 2. 推荐的 LaTeX 层级（直接决定“像不像顶刊”）

### 2.1 基础骨架（Cell/Nature Reviews 兼容）

建议你的 `{主题}_review.tex` 至少包含这些一级 section（名字可中英文二选一）：

- `Summary` / `Abstract`（或 `摘要`）
- `Introduction`（或 `引言`）
- 主体 taxonomy：4–7 个 `\section{...}`（用概念命名，而不是“步骤/编号/项目符号”）
- `Questions and concerns`（或 `讨论`，承担“批判性综合/局限/争议”的功能）
- `Perspectives`（或 `结论` / `Outlook`，承担“研究议程/转化落地”的功能）

> 注：本仓库的 pipeline 校验脚本已兼容中英文标题（见 `systematic-literature-review/scripts/validate_review_tex.py`）；你可以用 `Summary/Introduction/Questions and concerns/Perspectives` 这类顶刊常见标题，不必为了通过校验而刻意塞中文关键词。

### 2.2 反例（常见“像笔记”的结构）

- 一级标题过多（>10）且全是并列列表
- 没有“争议/局限”集中收束段
- 结尾只有“未来展望一句话”，没有可检验议程

## 3. 可直接套用的 `{主题}_review.tex` 模板（推荐）

以下模板以 **LaTeX + BibTeX（bst）** 为准，引用统一用 `\cite{key}`（并确保 `{主题}_参考文献.bib` 存在对应 key）。

> 你可以把下面这份骨架当作“可编译占位稿”，然后逐段把占位符替换为真实内容。

```tex
\documentclass[11pt,a4paper]{article}

\usepackage[margin=2.5cm]{geometry}
\usepackage{parskip}
\usepackage{graphicx}
\usepackage{booktabs}
\usepackage{longtable}
\usepackage{hyperref}

% 中文支持（如全文英文可删除）
\usepackage{ctex}

% References: BibTeX + bst
\usepackage{natbib}
\bibliographystyle{gbt7714-nsfc} % 可替换为其他 bst
\bibliography{{主题}_参考文献}

\title{{主题}}
\author{Author Name(s)}
\date{\today}

\begin{document}
\maketitle

\begin{abstract}
% Summary/Abstract：1 段为主（必要时 2 段），先给中心主张，再给争议/新机制/新框架，最后给“本文将如何组织证据”。
【摘要】{一句话说明研究对象/重要性}。{一句话给出“旧共识”}。{一句话指出“新数据/新方法”带来的冲突或扩展}。{一句话给出本文中心主张（可被反驳）}。{一句话说明本文将用怎样的 taxonomy 组织证据，并点出最重要的边界条件}。
\end{abstract}

\section{引言}
% 引言三段法：旧共识 → 新数据/争议 → 本综述目标与路线图（最后一段必须明确）
{段落 1：为什么重要 + 旧共识是什么 + 旧共识带来的默认解释}\cite{key}.

{段落 2：新数据/新方法改变了什么 + 为什么引发争议 + 现有综述的缺口在哪里}\cite{key1,key2}.

{段落 3（路线图）：本文聚焦回答 2–4 个具体问题，并按 taxonomy 组织为 N 个主题；最后一句明确本文的“边界”（不覆盖什么）}\cite{key}.

\section{主体主题 1：{一级主题名（概念命名）}}
% 主题开头：先给结论/立场，再用证据链支撑，末尾给边界与反例
{Topic sentence：该主题在整篇稿件中的角色（解释哪类不一致/失败模式）}\cite{key}.

\subsection{子主题 1.1：{子主题名}}
{机制/原理层解释（尽量可检验）}\cite{key}.
{关键证据 1（给出研究类型/样本/指标/效应方向）}\cite{key}.
{关键证据 2（独立证据或对照/消融）}\cite{key}.
{边界/反例：在哪些条件下不成立、失败模式是什么}\cite{key}.

\subsection{子主题 1.2：{子主题名}}
{同上：主张-证据-边界}\cite{key}.

\section{主体主题 2：{一级主题名}}
{同样结构：主题角色 → 子主题展开 → 边界/争议}\cite{key}.

% ……（主体主题 3–6，按你的 Taxonomy 覆盖清单来）

\section{讨论：Questions and concerns（方法学与争议收束）}
% 这一节的功能：把“为什么会不一致/为什么会争议/为什么可能是假阳性”集中讲清楚
\subsection{证据异质性的来源（可归因、可验证）}
{把异质性归因到：定义/阈值/平台/队列/数据集/实现细节/对照不足等}\cite{key}.

\subsection{方法学红线与常见陷阱}
{列出 4–6 条红线：比如数据泄漏/选择偏倚/批次效应/未注册/报告透明度不足/对照不充分，并指出如何在未来研究中避免}\cite{key}.

\subsection{仍未解决的关键问题（必须可检验）}
{用 3–6 条 bullet-like 句子描述：每条都能对应到“需要怎样的实验/数据/系统”}\cite{key}.

\section{结论：Perspectives（研究议程与可落地含义）}
% 结论不是“再总结一遍”，而是给出可执行的研究路线与现实约束
{一句话回收中心主张，并说明其成立的关键边界}\cite{key}.

\subsection{研究议程（5–10 条可检验问题）}
{把议程写成可检验问题，并标注需要的证据类型：多中心/外部验证/跨平台/真实世界等}\cite{key}.

\subsection{实践落地与标准化（按领域改写）}
{临床：伴随诊断/监管/阈值迁移/成本-周转；CS：算力/延迟/可扩展性/开源复现；工程：成本/标准/可靠性/产业化}\cite{key}.

\section*{Acknowledgments}
{感谢资助/协作/数据来源（如需要）}.

\section*{Declaration of interests}
{声明利益冲突（如无：The authors declare no competing interests.）}.

\bibliographystyle{gbt7714-nsfc}
\bibliography{{主题}_参考文献}
\end{document}
```

## 4. 各 section 的“写作动作”模板（更细颗粒度）

### 4.1 Summary / Abstract（顶刊信号最强的一段）

**目标**：在 150–250 英文词（或约 300–450 中文字）内完成四件事：

1. **重要性**：为什么值得读
2. **旧共识**：过去大家怎么理解
3. **新变化**：新数据/新技术/新发现带来了什么冲突或扩展
4. **本文主张**：你支持哪种解释框架（可被反驳），以及你将如何组织证据

**可直接套用句式**（把 `{}` 换成你的内容）：

- `{对象/问题}` 长期被认为 `{旧共识}`，其主流解释依赖于 `{关键机制/关键假设}`\cite{key}.
- 然而，近期 `{新方法/新证据类型}` 的系统性发现显示 `{新现象/新类别/新规律}`，使得 `{争议/困惑}` 成为核心问题\cite{key1,key2}.
- 本文提出：`{中心主张（可证伪）}`；我们将以 `{taxonomy 的一级主题列表（逗号分隔）}` 组织证据，并在 `{边界条件}` 下讨论其可迁移性与失败模式\cite{key}.

### 4.2 Introduction（让审稿人知道你“不是在写教科书”）

**三段法动作**：

- 段 1：定义对象 + 旧共识（给出“为什么过去这样认为”的证据锚点）
- 段 2：新数据/争议出现（指出“为什么现在不够用了”）
- 段 3：本文目标与路线图（必须是具体问题；必须写边界）

### 4.3 主体 taxonomy 的每个一级主题（\section）的开头段

**开头段的最低要素**（缺一就容易变成“笔记堆砌”）：

- 这一主题在整篇稿件中的角色（它解释哪类不一致/失败模式）
- 你在这一主题下要 defend 的“局部主张”
- 你准备拿哪些证据类型来 defend（临床试验/系统发现/机制实验/基准评测…）

**模板**：

`{主题名}` 是理解 `{总体主张}` 的关键，因为它决定了 `{一类不一致/失败模式}` 如何产生。我们主张 `{局部主张}`，并基于 `{证据类型 A}` 与 `{证据类型 B}` 的一致性证据展开，同时指出在 `{边界条件}` 下该解释会失败\cite{key}.

### 4.4 Questions and concerns（把“批判性综合”集中到一处）

这一节建议至少覆盖三类问题（每类给 1–2 段）：

1. **定义与阈值问题**：关键术语/阈值/终点/指标如何导致结论漂移？
2. **方法学问题**：偏倚、对照不充分、数据泄漏、批次效应、复现性要素缺失
3. **解释框架问题**：哪些现象可能只是“伴随相关”，而非“因果机制”？

### 4.5 Perspectives（“研究议程 + 落地约束”）

顶刊 Review 的结尾通常不是“总结”，而是**给审稿人一个可验收的未来清单**：

- 5–10 条“可检验问题”（每条都能映射到所需证据类型）
- 1 段“落地约束”（临床/工程/CS 各有不同）
- 1 段“标准化路线”（定义/数据/协议/报告规范）

## 5. 快速自检（写完马上扫一遍）

- [ ] 每个 `\section{主体主题}` 的开头段是否有“角色 + 局部主张 + 证据类型 + 边界”？
- [ ] 是否有独立的 `Questions and concerns`（或等价功能段）？
- [ ] `Perspectives` 是否给出 5–10 条可检验研究问题？
- [ ] 关键主张是否都有 `\cite{key}` 支撑，且 `.bib` 中存在对应 key？
- [ ] 标题是否以“概念/问题”命名，而不是“步骤/编号/项目符号”？

> 写作提示：正文需满足 `config.yaml` 中 `writing.min_total_words/min_body_sections/min_section_words` 的硬门槛。若未达标，补充内容时优先增加具体案例、证据链、边界与反例，避免空话凑字。

> 数据来源说明：默认正文基于检索元数据生成，未自动下载 PDF 全文。若需全文补强，优先对关键引用下载/阅读，再增加方法细节、效应量与局限描述。
> 若可获取摘要（默认从 OpenAlex 重组），写作时应充分使用摘要中的方法/结果/局限，不要只停留在标题层面。
