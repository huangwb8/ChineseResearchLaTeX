# 专家级综述写作指南（轻量版）

> 适用于 `systematic-literature-review` 新工作流：相关性评分→选文→自由写作。硬性指标仅限字数/参考数与章节存在性，其余依赖专家判断。

## 核心原则
- **不可偷懒/短视**：不为赶进度牺牲论证质量；不确定点要说明。
- **中心论点明确**：用 1–2 句提出可辩护的主张，再用证据链支撑。
- **叙述体证据链**：同段写"主张→证据→边界/反例→前瞻"，避免列表堆砌。
- **高分文献优先**：评分高的文献应成为核心证据，低分仅作背景需标注局限。
- **内容分离**：综述正文聚焦领域知识，方法学流程放在工作条件中。

## 内容分离原则（防止 AI 流程泄露）

### 为什么需要内容分离？

传统学术综述与 AI 生成的综述有一个关键区别：**人类作者不会在正文中描述自己是如何检索和筛选文献的**。这些方法学信息通常放在：
- 独立的"方法学"章节（对系统综述而言）
- 补充材料
- 或者完全不提及

如果综述正文中出现"本综述基于 400 条初检文献..."或"方法学上，本综述按照'检索→去重→评分→选文→写作'的管线执行"，读者会立即感受到"这是 AI 写的"，破坏阅读体验和专业性。

### 禁止在综述正文出现的内容

**❌ 禁止模式（AI 流程泄露）**：

| 位置 | 禁止内容示例 | 问题 |
|------|-------------|------|
| **摘要** | "本综述基于 400 条初检文献、去重后 285 条、最终保留 150 篇代表性研究" | 泄露检索流程统计 |
| **摘要** | "我们采用相关性评分系统从 285 篇文献中筛选出 150 篇高相关性研究" | 泄露评分选文流程 |
| **引言** | "方法学上，本综述按照'检索→去重→相关性评分→高分优先选文→写作与校验'的管线执行" | 泄露完整工作管线 |
| **引言** | "文献检索采用多查询策略，在 OpenAlex 数据库中获取候选文献" | 泄露检索细节 |
| **任何章节** | "我们使用了字数预算系统来平衡各章节的内容分配" | 泄露写作工具 |

### 允许的替代表述

**✅ 正确模式（聚焦领域知识）**：

| ❌ 错误（流程泄露） | ✅ 正确（领域知识） |
|------------------|------------------|
| "本综述基于 400 条初检文献、去重后 285 条、最终保留 150 篇" | "本综述系统梳理了该领域的最新研究进展" |
| "我们采用相关性评分系统筛选出 150 篇高相关性研究" | "我们纳入了该领域最具代表性的研究" |
| "方法学上，本综述按照'检索→去重→评分→选文→写作'的管线执行" | （完全删除，不提方法学） |
| "文献检索采用多查询策略，在 OpenAlex 数据库中获取候选文献" | （放在工作条件中，不在正文提及） |

### 方法学信息应放在哪里？

**✅ 正确位置**：`{主题}_工作条件.md`

```markdown
## Search Log
- 检索策略：多查询检索，生成 8 组查询变体
- 数据源：OpenAlex API
- 初检文献：400 条
- 去重后：285 条

## Relevance Scoring & Selection
- 评分方法：AI 逐篇阅读标题和摘要，按 1-10 分评分
- 高分优先比例：60-80%
- 最终选文：150 篇
```

### 写作检查清单

在提交综述正文前，请检查：

- [ ] 摘要中没有"基于 X 条文献"、"去重后 Y 条"、"最终保留 Z 篇"等描述
- [ ] 引言中没有"方法学上，按照 XXX 管线执行"等描述
- [ ] 任何章节都没有提及"检索"、"去重"、"评分"、"选文"、"字数预算"等元操作
- [ ] 综述完全聚焦于**领域知识**（研究背景、方法、结果、讨论、展望）
- [ ] 读者无法从正文中判断这是否由 AI 生成

- "基于 X 条文献"、"去重后 Y 条"等统计描述
- "按照 XXX 管线执行"等流程描述

## 推荐结构（固定）

### 摘要格式说明（重要）

- **格式**：**必须为单一段落**，禁止分段或使用项目符号
- **字数**：200–250 字
- **内容结构**：背景 → 核心发现/趋势 → 挑战 → 展望
- **禁止**：❌ 出现"本综述基于 X 条文献"、"最终保留 Z 篇"等 AI 流程泄露描述

> **为什么摘要必须是单一段落？**
>
> 学术综述的摘要遵循传统学术写作惯例，应作为一个连贯的整体呈现核心论点。分段或项目符号会破坏摘要的连贯性，让读者感受到"这是 AI 生成的列表"，而非专家撰写的论述。单一段落迫使 AI 在有限篇幅内进行逻辑整合，而非罗列要点。

### 固定章节
1. 摘要（详见上方格式说明）
2. 引言：问题重要性、研究空白、综述目标
3. 子主题段落（数量自定）：每段需包含主张、关键证据、局限/反例、未来方向
4. 讨论：异质性/失败原因/方法学或应用层面的挑战
5. 展望：可检验的研究议程、潜在突破口
6. 结论：收束论点与实践意义

## 写作要点
- **证据描述**：给出研究类型、样本量/数据集、主要结果方向；能量化尽量量化。
- **边界与风险**：说明适用范围、数据偏差、泛化/部署障碍。
- **多样性**：在子主题内交叉使用不同来源/时间段/方法，避免单一来源垄断。
- **未来工作**：提出具体可验证的假设或实验/数据/工程路径。
- **尊重元数据与事实**：整体润色仅做衔接/顺序/句式调整，不得改动文献题目/摘要所含事实、结果方向、数字、样本量或凭空添加新结论。
- **缩略词规范**：专有名词首次出现时采用"中文（英文全称，英文缩写）"格式，后续可直接使用英文缩写。
  示例："免疫检查点抑制剂（Immune checkpoint inhibitor，ICI）"；"卷积神经网络（Convolutional Neural Network，CNN）"。
  常见缩略词如"DNA"、"RNA"、"CT"、"MRI"、"AI"等可直接使用，无需首次全称展开。

## 单篇引用优先（核心原则）

### 为什么优先单篇引用？

- **明确来源**：每个观点都有明确的文献来源，读者可以追溯
- **自然节奏**：符合"引用 + 阐述 + 再引用 + 再阐述"的论述节奏
- **避免模糊**：避免"多项研究表明"这种模糊表述，让读者无法识别具体观点来源
- **提升可读性**：单篇引用让论述更清晰，逻辑更连贯

### 单篇引用的标准模式

**1. 引用具体方法/结果**（最常见，约占 50%）
```latex
Zhang 等人使用 ResNet-50 在 2000 张乳腺超声图像上达到 95\% 准确率\cite{Zhang2020}。
```

**2. 逐篇对比研究**（约占 15%）
```latex
ResNet 在乳腺超声分类中表现出色\cite{He2016}。随后，DenseNet 通过密集连接进一步提升了性能\cite{Huang2017}。相比之下，EfficientNet 降低了计算成本\cite{Tan2019}。
```

**3. 引用核心观点/理论**（约占 5%）
```latex
注意力机制能够帮助模型聚焦于关键区域\cite{Wang2021}。
```

### 何时使用小组引用（2-4篇）？

**仅限以下场景**（约占 25%）：

**1. 对比并列研究时**（且需明确说明各文献的差异化贡献）
```latex
在计算效率方面，EfficientNet 优于 ResNet\cite{Tan2019,He2016}，其中 Tan 等人采用复合缩放策略，而 He 等人使用残差连接。
```

**2. 引用互补证据时**（且需分别说明各文献的独立贡献）
```latex
多项研究表明，深度学习在乳腺超声分类中表现优异\cite{Zhang2020,Li2021}。Zhang 等人基于 ResNet-50，而 Li 等人采用 DenseNet，两者均达到 90\% 以上准确率。
```

### 禁止模式（必须避免）

**❌ 错误模式 1：模糊的"多项研究表明"**
```latex
多项研究表明 ResNet 表现优异\cite{Paper1,Paper2,Paper3}。
```
**问题**：读者无法识别每个观点的具体来源

**❌ 错误模式 2：堆砌文献无阐述**
```latex
深度学习在医学影像中取得进展\cite{A,B,C,D,E}。
```
**问题**：文献堆砌，没有具体阐述

**❌ 错误模式 3：单次引用 >4 个 key**
```latex
多种 CNN 架构被广泛应用\cite{Paper1,Paper2,Paper3,Paper4,Paper5,Paper6}。
```
**问题**：引用过多，读者无法识别每个文献的贡献

### 正确模式（优先单篇）

**✅ 正确模式 1：单篇引用 + 具体阐述**
```latex
Zhang 等人使用 ResNet-50 在 2000 张乳腺超声图像上达到 95\% 准确率\cite{Zhang2020}。Li 等人采用 DenseNet 进一步提升了性能\cite{Li2021}。Wang 等人引入注意力机制，使模型能够聚焦于关键区域\cite{Wang2022}。
```
**分析**：3 个单篇引用，每个都有具体阐述

**✅ 正确模式 2：小组引用 + 明确差异**
```latex
在计算效率方面，EfficientNet 优于 ResNet\cite{Tan2019,He2016}，其中 Tan 等人采用复合缩放策略减少了参数量，而 He 等人使用残差连接提升了训练稳定性。
```
**分析**：小组引用，但明确说明了各文献的差异化贡献

## 引用位置约束（引用紧跟观点，避免段末堆砌）

### 核心原则

**引用应紧跟着它所支持的观点**，而非堆积在段落末尾。

### 正确模式（推荐）

**✅ 模式 1：观点后立即引用**
```latex
ResNet 在乳腺超声分类任务中表现出色\cite{He2016}。Zhang 等人使用 ResNet-50 在 2000 张图像上达到 95\% 准确率\cite{Zhang2020}。Li 等人引入注意力机制进一步提升了性能\cite{Li2021}。
```

**✅ 模式 2：对比时逐个引用**
```latex
在计算效率方面，EfficientNet 优于 ResNet\cite{Tan2019}，其中 Tan 等人采用复合缩放策略。相比之下，He 等人使用残差连接提升了训练稳定性\cite{He2016}。
```

**✅ 模式 3：段末总结时偶尔堆砌（<20% 情况）**
```latex
前述研究表明，注意力机制在医学影像中具有重要价值\cite{Wang2021,Li2022,Zhang2023}。
```

### 禁止模式

**❌ 错误：段落末尾一次性堆砌所有引用**
```latex
深度学习在乳腺超声分类中取得了显著进展。多种 CNN 架构被广泛应用，包括 ResNet、DenseNet、EfficientNet 等。注意力机制也被引入以提升模型性能。迁移学习在小样本场景下表现出色\cite{Paper1,Paper2,Paper3,Paper4,Paper5,Paper6,Paper7,Paper8}。
```

**问题**：
- 读者无法判断哪个观点对应哪篇文献
- 违背学术写作基本规范
- 让读者感觉是"文献堆砌"而非"综合评述"

### AI 写作指南

写作时遵循以下节奏：
1. 提出观点/方法/结果
2. 立即引用支撑文献（`\cite{key}`）
3. 继续下一个观点
4. 再次引用

**避免**：
- 先写完整个段落，最后一次性加所有引用
- 将多个不相关的观点堆积在一起，然后堆砌引用

## 引用分布约束（避免引用堆砌）

### 目标分布（基于人类学术写作习惯）

基于对高质量学术综述的分析，健康的引用分布应遵循以下比例：

| 引用类型 | 目标占比 | 说明 |
|---------|---------|------|
| **单篇引用**（1篇） | 约 70% | 最常见形式，每个观点配 1-2 篇核心文献 |
| **小组引用**（2-4篇） | 约 25% | 用于对比研究或并列证据 |
| **大组引用**（>4篇） | <5% | 仅限综述性陈述或广泛共识 |

### 禁止模式

- ❌ **"陈述观点 + 堆砌大量文献"**：如 `\cite{key1,key2,...,key13}`（13篇）
- ❌ **在单个段落中集中引用 10+ 篇文献**：读者无法识别每个观点的具体来源
- ❌ **"\cite{key1,key2,...,keyN}" 后无具体阐述**：这种"文献堆砌"违背了"专家级人类的自然论述"原则

### 推荐模式

- ✅ **"引用 + 具体观点阐述 + 再引用 + 再阐述"**的自然交替节奏
- ✅ **每个观点配 1-2 篇核心文献**，必要时追加对比文献
- ✅ **明确说明各文献的差异化贡献**，避免模糊的"多项研究表明"

### 写作技巧

**1. 按证据强度分层引用**
- 核心主张 → 1-2 篇高分文献（9-10分）：`\cite{HighScorePaper1}`
- 支持证据 → 2-3 篇中等文献（7-8分）：`\cite{MediumPaper1,MediumPaper2}`
- 对比/背景 → 3-5 篇低分文献（5-6分）：`\cite{Background1,Background2,Background3}`

**2. 按主题分段**
- 不要在一个段落内堆砌多个子主题的文献
- 每个子主题独立成段，各段落引用 2-5 篇相关文献
- 如果某子主题文献过多（>10篇），拆分为多个子段落

**3. 引用密度校验**
- 每段（约200-300字）建议引用 2-5 篇文献
- 避免"零引用段落"（无证据支撑）或"超载引用段落"（>10篇）
- 使用 `scripts/validate_citation_distribution.py` 检查分布

**4. 示例对比**

❌ **错误模式**（引用堆砌）：
```latex
深度学习在乳腺超声分类中取得了显著进展\cite{Paper1,Paper2,Paper3,Paper4,Paper5,Paper6,Paper7,Paper8,Paper9,Paper10,Paper11,Paper12,Paper13}。多种CNN架构被广泛应用。
```

✅ **正确模式**（自然交替）：
```latex
ResNet 在乳腺超声分类任务中表现出色，Zhang 等人使用 ResNet-50 在 2000 张图像上达到 95% 准确率\cite{Zhang2020}。随后，Li 等人引入注意力机制进一步提升了性能\cite{Li2021}。与此同时，DenseNet 因其参数效率高而备受关注\cite{Wang2022}。
```

### 验证工具

使用 `scripts/validate_citation_distribution.py` 检查你的引用分布：

```bash

```

输出示例：

## 引用多样性约束（避免引用不均）

### 为什么需要引用多样性？

引用分布不均衡会严重影响综述质量：

- **零引用段落**：缺乏证据支撑，读者无法验证观点来源
- **引用集中**：某段落引用过多，其他段落引用不足，导致论证不平衡
- **未引用文献过多**：可能意味着选文阶段噪声偏高或摘要缺失过多；但**不要求用完所有文献**（低分/边缘背景/摘要缺失文献可以不引用）
- **高频文献占比高**：过度依赖少数几篇文献，无法体现全面调研

### 目标指标

| 指标 | 目标阈值 | 检测内容 |
|------|---------|---------|
| **零引用段落率** | <10% | 识别无引用支撑的段落 |
| **段落引用密度方差** | <3 | 检测引用分布不均 |
| **文献利用率（仅提示）** | 无硬性阈值 | 检测未被引用的文献（用于回溯选文噪声，而非逼迫“用完”） |
| **高频文献占比** | <15% | 检测过度引用（被引用≥5次的文献占比） |

### 验证工具

启用引用多样性检测（需要提供 BibTeX 文件）：

```bash
python scripts/validate_citation_distribution.py \
  review.tex \
  --check-diversity \
  --bib references.bib \
  --verbose
```

输出示例：
```
🌐 引用多样性检测:
  零引用段落率:       0.0% (0/20)  [目标: <10%]  ✓
  段落引用密度方差:  1.45  [目标: <3]  ✓
  平均每段引用数:     2.8
  文献利用率:        92.3% (120 篇被引用)  [提示项：不做硬性门槛]
  高频文献占比:       8.5% (12 篇被引用≥5次)  [目标: <15%]  ✓

🎉 多样性总体结果: ✓ 通过
```

### 改进建议

#### 零引用段落率过高
- **问题**：当前 40%，共 5 段无引用
- **建议**：每个段落至少引用 1-2 篇文献以提供证据支撑
- **操作**：为零引用段落添加相关文献引用

#### 段落引用密度分布不均
- **问题**：方差 4.5，目标 <3
- **建议**：平衡各段落的引用数量，避免某段过度引用而其他段落引用不足
- **操作**：将引用过多的段落的文献拆分到其他相关段落

#### 文献利用率偏低
- **问题**：当前 60%，仅 90 篇被引用（BibTeX 中有 150 篇）
- **建议**：不要为“用完文献”而强行加引用；优先回到选文阶段减少噪声（移除低分/摘要缺失/边缘背景文献），或将真正相关的证据按观点拆分后自然引用
- **操作**：
  - 若未引用文献主要是低分/摘要缺失：允许不引用（不作为失败）
  - 若未引用文献明显无关：从 BibTeX 文件中删除（降低噪声与后续对齐成本）
  - 若未引用文献其实重要：把它们对应的观点写出来，而不是写“补充阅读可参见”

#### 高频文献占比过高
- **问题**：当前 25%，有 30 篇被引用≥5次
- **建议**：避免过度依赖少数几篇文献，应增加引用的多样性以体现全面调研
- **操作**：引入更多元文献，平衡引用频次

### 最佳实践

1. **写作时**：
   - 每个段落至少引用 1-2 篇文献
   - 避免在某一段落中集中引用大量文献
   - 充分利用筛选出的高分文献

2. **校验时**：
   - 使用 `--check-diversity` 检测引用不均衡
   - 关注零引用段落列表，逐段补充引用
   - 检查文献利用率，决定是否删除未引用文献

3. **优化时**：
   - 平衡各段落引用密度（目标方差 <3）
   - 不追求“用完所有文献”；优先保证证据链清晰、引用紧贴观点
   - 控制高频文献占比 <15%（体现全面调研）

## 写作负面约束（禁止模式）

### 为什么需要负面约束？

专家级人类作者在学术综述中**不会**做以下事情。如果 AI 生成这类内容，读者会立即感受到“这是业余水准/在凑字数”。

### 禁止模式 1：补充阅读/参见类句子

**❌ 严格禁止**：

```latex
本节补充阅读可参见：\cite{key1,key2,key3}。
进一步阅读可参考：\cite{key4,key5,key6}。
相关研究参见：\cite{key7,key8,key9}。
```

**为什么业余**：
- 对读者没有价值：读者不知道这些文献具体贡献了什么
- 典型的“凑字数”行为：专家级综述不会这样堆砌文献
- 违反引用紧贴原则：引用应该紧跟着它所支持的观点

**正确做法**：
- 如果这些文献确实重要：在段落中逐个引用并说明其贡献
- 如果它们不够重要：就不要引用（低分文献可以不引用）

### 禁止模式 2：模糊的引用堆砌

**❌ 严格禁止**：

```latex
多项研究表明深度学习有效\cite{key1,key2,key3,key4,key5}。
```

**为什么业余**：
- 读者无法识别每个观点的具体来源
- 这是“文献堆砌”，而非“综合评述”

**✅ 正确模式**（引用 + 阐述 + 再引用）：

```latex
Zhang 等人使用 ResNet-50 达到 95\% 准确率\cite{Zhang2020}。Li 等人引入注意力机制进一步提升了性能\cite{Li2021}。相比之下，DenseNet 降低了计算成本\cite{Wang2022}。
```

### 禁止模式 3：为达到字数而灌水

**❌ 严格禁止**：
- 添加无实质内容的过渡句
- 重复表述同一观点
- 为“用完”所有文献而强行引用低分文献

**✅ 正确做法**：
- 如果段落完整但字数不足：补充具体证据、数字、反例（有机扩写）
- 如果高分文献已充分引用：可以不引用低分文献
- 如果确实需要补充背景：拆分为独立子段落，每段 2-5 篇文献

### 写作检查清单（交付前自检）

- [ ] 没有“补充阅读可参见”“进一步阅读可参考”“相关研究参见”等句子
- [ ] 没有“多项研究表明\cite{key1,key2,key3}”这种模糊堆砌（除非前文已逐个引用过）
- [ ] 单次引用不超过 6 个 key（段末总结除外）
- [ ] 低分文献（<5 分）没有被强行引用
- [ ] 每个引用都紧跟着它所支持的观点

## 常见风险与修正
- **引用堆砌**：单次 \cite{} 包含 >5 个 key，需拆分为多个独立的引用陈述。
- **引用不均衡**：若某段引用稀少，补充高分文献或合并到更强的主题。
- **段落空洞**：若缺少结果/数字，返回评分库查找能支撑的证据；注明"需全文核实"。
- **跑题**：检查每段是否直接支撑中心论点；无关段落删减或移至背景。

## 交付前自检（静态）
- 章节齐全：摘要、引言、≥1 子主题段落、讨论、展望、结论。
- 字数/参考数在目标范围；`\cite{}` 与 Bib key 对齐，无缺失。
- 每个子主题至少 1–2 条可核查的证据，并标注局限或反例。
