# 专家级综述写作指南

> 面向 `systematic-literature-review` 的正文写作。这里只保留硬规则与高频失误，避免把整套写作理念写成冗长说明。

## 核心原则

- 中心论点明确：每个主题段都要有可辩护主张
- 高分文献优先：低分文献只作背景或反例
- 内容分离：工作流信息只写进 `{主题}_工作条件.md`
- 证据链写法：主张 → 证据 → 边界/反例 → 前瞻

## 禁止流程泄露

正文禁止出现：

- “本综述基于 X 条文献”
- “去重后 / 最终保留 / 评分筛选”
- “按照检索→去重→评分→选文→写作的流程”
- “字数预算系统”

这些内容只能写进工作条件文档。

## 固定结构

1. 摘要
2. 引言
3. 子主题段落
4. 讨论
5. 展望
6. 结论

## 摘要规则

- 必须为单段
- 约 200-250 字
- 顺序：背景 → 主要趋势/发现 → 挑战 → 展望
- 禁止方法学流水账

## 引用规则

- 引用应紧跟观点，不要段末一次性堆砌
- 单篇引用优先；小组引用只用于并列比较
- 单次引用超过 4 篇应视为异常
- “多项研究表明\cite{...}” 只有在前文已逐个展开时才勉强可用

## 每个子主题段最少包含

- 局部主张
- 1-3 条关键证据
- 适用边界或反例
- 下一步问题或方向

## 讨论与展望

- 讨论集中处理异质性、失败原因、方法学陷阱和争议来源
- 展望必须给出可检验研究议程，不是空泛愿景

## 缩略词

- 首次出现用“中文（英文全称，缩写）”
- DNA / RNA / CT / MRI / AI 这类常见缩写可直接使用

## 快速自检

- [ ] 正文没有流程泄露
- [ ] 摘要是单段
- [ ] 子主题不是文献堆砌
- [ ] 引用靠近对应观点
- [ ] 讨论与展望不是重复总结
