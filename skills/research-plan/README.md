# Research Plan

这个 skill 用来围绕一个科研问题做文献调研、方法提炼和执行计划设计，适合“先研究顶级论文，再反推出适合自己项目的分析路线”的场景。

## 旧名兼容

- 当前正式名：`research-plan`
- 旧名：`make-research-plan`
- 兼容状态：旧名 prompt 暂时仍会触发本 skill；安装新版本时系统级旧目录会被清理，不再保留 wrapper 目录。
- 历史工作区：`.make-research-plan/` 保持不变，这是稳定产物目录，不等同于 skill 新名称。

## 用法

### 最推荐用法

```text
请使用 research-plan skill 为这个科研问题制定可执行的分析策略。
输入：研究问题描述，以及可选的工作目录
输出：文献调研结果、方法总结和最终分析计划
```

### 进阶用法

```text
请使用 research-plan skill 为这个课题制定研究计划。
输入：研究问题描述，工作目录 `./project`
输出：文献调研、方法框架和最终计划
另外，还有下列参数约束：
- 优先近 5 年高质量论文
- 重点关注可复现的方法细节
- 输出尽量能直接转成实验路线
```

## 能做什么

- 从研究问题里提炼主题、关键词和相关方法方向。
- 组织文献调研，并把结果沉淀到隐藏工作目录。
- 提炼方法模式、分析框架和最终执行计划。
- 适合实验设计、数据分析流程规划、技术路线梳理。
- 不适合代替正式实验执行，也不是简单的“搜几篇论文标题”工具。

## 使用示例

### 示例 1：为一个新课题做方法调研

```text
请使用 research-plan skill 为这个科研问题制定计划。
输入：`如何基于多组学数据构建可解释预测模型`
输出：研究主题、关键方法和执行计划
```

### 示例 2：指定工作目录

```text
请使用 research-plan skill 为这个问题做文献驱动规划。
输入：研究问题描述，工作目录 `./proposal`
输出：`.make-research-plan/` 工作区和最终计划
```

### 示例 3：强调方法学提炼

```text
请使用 research-plan skill 为这个研究方向制定方案。
输入：研究问题描述
输出：分析计划
另外，还有下列参数约束：
- 重点比较不同论文的方法框架
- 输出要包含推荐步骤和潜在风险
```

## 输出

- 默认会在工作目录下创建 `.make-research-plan/` 隐藏工作区。
- 常见产物包括：
  - `metadata/theme.json`
  - `metadata/search_history.json`
  - `extracted/papers_info.json`
  - `analysis-framework.md`
  - `plan.md`
- 这个 skill 的价值不只是“找论文”，而是把论文读法、方法线索和执行计划串起来。

## 配置

- 配置文件：`research-plan/config.yaml`
- 默认工作目录名：`.make-research-plan`
- 关键配置节：
  - `work_directory.subdirs`
  - `work_directory.files`
- 目前更偏 Prompt 驱动，辅助脚本用于初始化和校验工作目录。

## 备选用法（脚本/硬编码）

如果你想先创建标准工作目录，再让 AI 继续推进，可以先运行初始化脚本。

### 初始化工作目录

```bash
python3 research-plan/scripts/initialize.py .
```

## 常见问题

### Q：这个 skill 会真的替我做实验吗？

A：不会。它的职责是“基于文献调研设计可执行路线”，不是直接跑实验。

### Q：为什么要创建 `.make-research-plan/`？

A：因为调研过程会产生主题提取、搜索历史、方法提炼和最终计划等中间产物，集中放在隐藏工作区更利于复盘。

### Q：如果我已经知道一些候选方法，还值得用吗？

A：值得。你可以把既有方向作为输入，让它帮你补充文献证据、比较方法优缺点和落地步骤。

### Q：它更适合哪类输出？

A：最适合“研究策略、实验设计、分析流程、技术路线”这类需要证据支撑的规划输出。
