## 总体变化概述

### 2025版 → 2026版 核心变化

| 变化类型 | 2025版 | 2026版 | 影响 |
|----------|--------|--------|------|
| **板块结构** | 3大板块 | 4大板块 | ⚠️ 高影响 |
| **章节编号** | 数字编号（1.1, 1.2） | 汉字+数字混合（（一）/1.1） | ⚠️ 中影响 |
| **风险应对** | 分散在"可行性分析"中 | 独立成小节 | ⚠️ 中影响 |
| **研究基础** | 作为"研究内容"的一部分 | 独立成一级板块 | ⚠️ 高影响 |

### 变化原因分析（政策背景）

1. **板块重组**: 2026版强调"研究基础与工作条件"的独立性，突出申请人的科研能力
2. **风险应对显性化**: 要求申请人明确列出研究风险，体现风险意识
3. **编号体系统一**: 采用更规范的编号格式，符合正式文档标准

### 2025版板块结构

```
一、立项依据与研究内容
├── 1.1 项目的立项依据
├── 1.2 研究内容
├── 1.3 研究目标
├── 1.4 拟解决的关键科学问题
├── 1.5 研究方案
│   ├── 1.5.1 研究方法
│   ├── 1.5.2 技术路线
│   └── 1.5.3 可行性分析
└── 1.6 特色与创新

二、研究基础与工作条件
├── 2.1 研究基础
└── 2.2 工作条件

三、年度研究计划与预期成果
├── 3.1 年度计划
└── 3.2 预期成果
```

### 2026版板块结构

```
（一）立项依据与研究内容
├── 1. 项目的立项依据

│   └── 2.3 关键科学问题
├── 3. 研究方案与可行性
│   ├── 3.1 研究方案
│   ├── 3.2 可行性分析
│   └── 3.3 研究风险应对 ← 新增
└── 4. 本研究的特色与创新

（二）研究基础与工作条件 ← 独立成一级板块
├── 1. 研究基础
├── 2. 工作条件
└── 3. 正在承担的项目 ← 新增

（三）年度研究计划与预期成果
├── 1. 年度计划

（四）其他 ← 新增板块

└── 2. 承担项目与完成情况
```

**拆分示例**:

```latex

\subsection{研究方法}

...
\subsection{可行性分析}

% 拆分为 2026版两个文件：
% extraTex/3.1.研究方案.tex（包含：研究方法、技术路线）
% extraTex/3.2.可行性分析.tex（包含：可行性分析）
```

### 多对一映射（需要合并）

**无此类映射**（2025→2026主要是拆分，很少合并）

### 新增章节（需要生成）

| 2026版新增章节 | 来源建议 | 生成方法 |
|----------------|----------|----------|

| `extraTex/（四）2.承担项目与完成情况.tex` | 无对应内容 | 手动补充 |

**2025版无删除章节**（所有内容都有对应位置）

## 新增性能要求

### 2026版新增明确要求

1. **研究风险应对**（3.3节）
   - **要求**: 明确列出技术风险、进度风险、资源风险
   - **内容**: 风险识别 + 应对措施 + 预案
   - **字数建议**: 300-500字

2. **正在承担的项目**（（二）3节）
   - **要求**: 列出当前正在承担的科研项目
   - **内容**: 项目名称、资助号、资助金额、起止时间、本人角色
   - **格式**: 表格或列表

3. **申请人简历**（（四）1节）
   - **要求**: 简要学术经历
   - **内容**: 教育背景、工作经历、学术兼职
   - **字数建议**: 500-800字

4. **承担项目与完成情况**（（四）2节）
   - **要求**: 既往项目完成情况
   - **内容**: 已完成项目、发表成果、人才培养
   - **格式**: 表格优先

## 格式与样式变化

### 编号体系变化

**2025版**:
```latex
\section{（一）立项依据与研究内容}
\subsection{1.1 项目的立项依据}
\subsubsection{1.1.1 研究意义}
```

**2026版**:
```latex
\section{（一）立项依据与研究内容}
\subsection{1. 项目的立项依据}  % 注意：数字后有圆点
\subsubsection{1.1 研究意义}    % 注意：不再有二级圆点
```

**迁移建议**:
- ⚠️ **不要手动修改章节编号**，由模板自动控制
- 只迁移 `\subsection{}` 内的内容文本
- 忽略原有的 `\section{}` 和 `\subsection{}` 标记

### 字数要求变化

| 章节 | 2025版字数 | 2026版字数 | 变化 |
|------|-----------|-----------|------|
| 立项依据 | 1500-2000 | 2000-2500 | +500 |
| 研究内容 | 800-1000 | 1000-1200 | +200 |
| 研究方案 | 1000-1500 | 1200-1500 | +200 |
| 可行性分析 | 500-800 | 800-1000 | +300 |
| 研究基础 | 1000-1500 | 1500-2000 | +500 |

**迁移建议**:
- 旧内容可能不足新字数要求，需要补充
- 在Phase 4迭代优化中自动补充内容

### 参考文献格式

**2025版**:
```latex
\cite{Author2024}  % 简单引用
```

**2026版**:
```latex
\cite{Author2024}  % 格式不变，但更严格检查
\citep{Author2024} % 新增：圆括号引用
\citet{Author2024} % 新增：文本引用
```

**迁移建议**:
- 检查所有 `\cite{}` 是否有对应的 `.bib` 条目
- 更新不规范的引用格式

## 迁移建议

### 推荐迁移策略

**对于 2025→2026 迁移，推荐使用 `smart` 策略**:

```yaml
strategy: "smart"
理由:
  - 存在一对多拆分（研究方案）
  - 存在新增章节（风险应对、正在承担的项目）
  - 板块结构有较大变化（研究基础独立）
```

### 关键迁移步骤

#### 步骤1: 准备阶段
1. 备份 2025 版项目
2. 创建 2026 版新项目（从模板复制）
3. 对比两个项目的 `main.tex` 结构

#### 步骤2: 简单一对一迁移
优先迁移以下章节（低风险）:
- `1.1.立项依据` → `1.立项依据`
- `1.2.研究内容` → `2.1.研究内容`
- `1.3.研究目标` → `2.2.研究目标`
- `1.4.关键科学问题` → `2.3.关键科学问题`
- `1.6.特色与创新` → `4.特色与创新`
- `2.1.研究基础` → `（二）1.研究基础`
- `2.2.工作条件` → `（二）2.工作条件`
- `3.1.年度计划` → `（三）1.年度计划`
- `3.2.预期成果` → `（三）2.预期成果`

#### 步骤3: 复杂一对多迁移
拆分 `extraTex/1.5.研究方案.tex`:
```python
# 伪代码
content = read("old/extraTex/1.5.研究方案.tex")

# 识别拆分点
split_marker = "\\subsection{可行性分析}"
scheme_part = content[:content.find(split_marker)]
feasibility_part = content[content.find(split_marker):]

# 写入新文件
write("new/extraTex/3.1.研究方案.tex", scheme_part)
write("new/extraTex/3.2.可行性分析.tex", feasibility_part)
```

#### 步骤4: 生成新增内容
- 调用 `nsfc-methods-feasibility-writer` 生成 `3.3.研究风险应对.tex`
- 手动补充 `（二）3.正在承担的项目.tex`
- 手动补充 `（四）1.申请人简历.tex`
- 手动补充 `（四）2.承担项目与完成情况.tex`

#### 步骤5: 修复引用
```bash
# 扫描所有 \ref{}, \cite{}
# 更新标签编号以匹配新结构
```

#### 步骤6: 迭代优化（5轮）
- 第1轮: 检查逻辑连贯性，添加过渡段
- 第2轮: 统一术语，修复引用
- 第3轮: 补充内容深度，增加证据
- 第4轮: LaTeX编译验证，修复格式
- 第5轮: 全文通读，最终润色

#### 步骤7: 最终验证
```bash
cd new_project
xelatex main.tex
bibtex main
xelatex main.tex
xelatex main.tex
```

### 常见问题与解决方案

#### Q1: 拆分"研究方案"后逻辑不连贯

**解决方案**:
```latex
% 在 extraTex/3.2.可行性分析.tex 开头添加过渡段：
综上所述，本研究采用\ref{sec:methods}所述方法与技术路线，具有良好的可行性。具体分析如下：
```

#### Q2: 新增"研究风险应对"不知道写什么

**解决方案**:
调用技能生成内容：
```
调用 nsfc-methods-feasibility-writer
输入：旧项目的"可行性分析"部分
输出：基于现有内容的风险识别与应对措施
```

#### Q3: 字数不足怎么办

**解决方案**:
- 在 Phase 4 迭代优化中自动补充
- 调用对应的写作技能（如 `nsfc-rationale-writer`）补强内容
