# 协作计划模板

当 `paper-write-sci` 以 `collaborative` 模式运行时，计划文件至少应包含以下栏目。

```markdown
# WritePaperSCI Plan

## Scope
- 论文主题：
- 模式：collaborative
- 目标风格：
- 目标文件范围：

## Global Assessment
- 主线是否清楚：
- 最大风险：
- 最值得先修的 3 个问题：

## Section-role Risk
- 最容易发生章节串位的位置：
- 哪些段落在重复 `Results`：
- 哪些段落缺少真正的解释工作：
- 哪些段落需要从“证据叙述”改写为“意义叙述”：

## Issues
### Issue 1
- 位置：
- 问题类型：事实 / 数字 / 逻辑 / 风格 / 结构 / 章节分工
- 证据：
- 为什么有问题：
- 建议动作：
- 关联图表：
- 风格锚点：
- 风险等级：

## Execution Order
1. 先修什么：
2. 再修什么：
3. 最后修什么：
```

## 使用原则

- 计划是给人类审查的交付物，不是内部草稿
- 计划只总结问题与建议，不直接改写全文
- 风险等级要让人一眼区分“阻塞问题”和“润色问题”
- 若任务涉及 `Discussion` 优化，优先显式指出“复述 Results”“缺少解释”“空心评论”和“应保留的少量定量锚点”
