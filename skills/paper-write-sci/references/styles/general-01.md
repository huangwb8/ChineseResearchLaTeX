# general-01

## 元信息

- 风格 ID：`general-01`
- 领域：未指定，适合作为通用 SCI 风格
- 适用场景：没有明确作者风格、跨学科、初稿修订、稳健润色
- 风格定位：基于官方写作建议抽象出的“清楚、可信、可发表”风格

## 风格总目标

1. 每一节都承担清楚的功能
2. 读者能快速把握研究问题、方法、主要发现和意义
3. 结果与讨论分工明确
4. Methods 足够可复现
5. 标题、摘要、图表引用和缩写都尽量简洁而准确
6. `.tex` 源文结构应让人容易从渲染结果回到对应证据点

## 核心原则

### 1. 清楚优先

- 用清晰胜过用“高级”
- 每段只推进一个主要任务
- 重要信息放句首或句尾

### 2. 结构优先

- 先搭章节功能，再优化句子
- 让 Introduction 提问题、Methods 交代做法、Results 报事实、Discussion 解释意义

### 3. 证据优先

- 关键论断要有文献或结果支撑
- 结果段落的数字和图表要一一对应
- 讨论中的推断不能超出结果材料

### 4. `.tex` 可回看优先

- 新段落才用空行
- 同一段内若存在多个证据点、多个比较点或多个 panel，优先一行一个“并列信息单元”
- 这里的“点”优先指并列证据单元、panel、比较项或局限项，不是任意一句话
- 不机械拆句，也不为了统一格式大面积重排现有正文
- 只作用于自然语言段落，不主动重排行敏感 LaTeX 结构

## 分章节写法

### Title

官方建议的共同点可以归纳为：

- 清楚而具体
- 尽量短，但不牺牲信息量
- 尽量让读者一眼看出研究对象、关键方法或主要发现

实践建议：

- 优先使用“对象 + 发现/作用 + 方法/场景”的结构
- 少用夸张形容词
- 不把摘要压缩成标题

### Abstract

推荐结构：

1. 研究背景或知识空白
2. 采用了什么方法或数据
3. 最重要的结果，最好带关键数字
4. 结果意味着什么

实践建议：

- 把最有信息量的结果放进去，而不是只写“显著改善”
- 不写与正文无关的新信息
- 若期刊限制字数，先保留问题、方法、关键结果、意义四件事

### Introduction

推荐逻辑：

1. 研究主题为什么重要
2. 目前知道什么
3. 目前还不知道什么，或者现有方法有什么限制
4. 本研究要解决什么问题
5. 本研究做了什么

实践建议：

- 用“漏斗式”逐步收束到研究问题
- 文献不要堆砌成书目摘要，要为提出问题服务
- 最后一段明确研究目的与贡献

### Methods

官方建议的共同落点是可复现性。

必须覆盖：

- 样本/数据来源
- 纳入排除标准
- 实验或分析流程
- 统计方法
- 软件、包、版本与关键参数
- 伦理信息（如适用）

实践建议：

- 按实际操作顺序组织，而不是想到什么写什么
- 用过去时为主
- 不把结果结论混进 Methods

### Results

通用高质量写法：

1. 每个小节开头交代这一段在回答什么问题
2. 简要说明使用的分析动作
3. 报告最关键的观察结果与数字
4. 用图表作为证据锚点

实践建议：

- Results 主要写“看到了什么”，不要过早写“为什么会这样”
- 子标题尽量信息化，而不是只写“Result 1”
- 负结果、边界结果也要诚实呈现
- 同一段连续报告多个结果点时，优先同段分点换行，让每行对应一个可直接定位的证据点

### Figure Legends

- 先说明图的主题，再按 panel 或判读动作展开
- 同一条 legend 中，panel、视觉编码说明、统计标记说明优先逐行展开
- 若各行仍共同服务于同一段落，就不要插空行

### Discussion

推荐结构：

1. 先概括最重要的发现
2. 解释这些发现与现有研究的关系
3. 说明研究的独特贡献
4. 讨论局限性
5. 指向下一步工作或应用意义

实践建议：

- Discussion 不是重复 Results
- 解释结果时要区分“结果支持的结论”和“合理但尚未证实的推测”
- 局限性要具体，不要用模板化套话轻轻带过
- 不要连续堆叠图号、检验名和多个效应值；默认每段只保留极少量、真正服务于解释的定量锚点
- `Discussion` 的具体性主要来自解释、文献定位、边界和转化意义，而不是再次汇报数字

### Conclusion

- 简短重申研究给出了什么新信息
- 强调边界清楚的意义
- 不引入新数据、新引用或新论点

## 语言特征

### 时态

- Introduction：现在时
- Methods：过去时
- Results：过去时
- Discussion：现在时与过去时混合，但要稳定

### 句子组织

- 单句不宜过载
- 长句中尽量只保留一层核心从属关系
- 用自然连接词，而不是堆砌转折词

### `.tex` 行级组织

- 段落边界靠空行表达
- 同段内的多个点靠换行表达
- 这里的“点”优先指并列证据单元或 panel，不是任意句子
- 一行最好只承载一个便于人类回源定位的点

### 语气

- 客观、克制、专业
- 结论可以有力度，但不夸大

## 数字与统计表达

- 百分比保留适度精度，通常 1 位小数足够
- 统计量格式全文统一
- 报数字时给读者必要的比较参照
- 不要只报 `p` 值而不报效果大小或实际差异

## 图表与证据

- 文中提到的关键结果应能在 Figure/Table 中找到对应
- 句中引用图表时要让图表承担证据角色，而不只是装饰引用
- 多图多表时按叙事顺序组织，而不是机械按编号堆叠

## 缩写与术语

- 首次出现先给全称
- “首次出现”按整篇论文所有正文 tex 联合判断，不按单个文件局部判断
- 只保留真正能提升可读性的缩写
- 同一概念全篇用同一说法

## 应该模仿什么

- 清晰的章节功能
- 有证据支撑的结果表达
- 克制但明确的讨论与结论
- 稳定的术语和数字格式

## 不应该模仿成什么

- “高大上但空泛”的大词堆砌
- 把标题写成广告语
- Abstract 只有背景没有结果
- Results 和 Discussion 混成一团
- 局限性只写一句无信息量的话

## 示例锚点

- 好的 Abstract：一两句背景后立刻给方法和关键结果
- 好的 Results：每节先说回答什么问题，再给证据和数字
- 好的 Discussion：先解释发现，再说明边界与局限

## 使用提醒

- 当用户没有明确作者风格时，用这个风格先把结构、清晰度和发表友好度做好
- 如果后续再叠加某个作者风格，应保留这里的“结构清楚、证据扎实、边界克制”底线
- 当同一段需要承载多个并列证据点时，默认用“同段分点换行”保持 `.tex` 可读性
