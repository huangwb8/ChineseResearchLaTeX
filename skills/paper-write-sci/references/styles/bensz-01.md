# bensz-01

## 元信息

- 风格 ID：`bensz-01`
- 领域：生物医学，尤其是肿瘤学、免疫学、生物信息学
- 适用场景：肿瘤免疫、分型建模、预测模型、转化医学论文
- 风格定位：不是“泛 SCI”，而是更像真实作者亲手写出来的论文

## 风格总目标

写作时优先追求以下体验：

1. 读者很快知道“问题是什么”
2. 作者提出的方法或概念有鲜明命名，便于记忆
3. 关键发现依赖具体数字和跨组对比，而不是空泛陈述
4. Figure legend 能让非领域读者独立读懂图里每个编码在表达什么
5. 局限性写得诚实、具体、不回避
6. 全文像一个人在持续讲同一个故事，而不是拼接模板段落
7. `.tex` 源文回看时，关键证据点和图注 panel 容易被直接定位

## 风格指纹

### 1. 问题导向开头

- 开头不先铺陈“大背景很重要”
- 先承认已有进展，再立刻指出实际空白
- 常见结构：`已有治疗/方法有效 -> 但只有部分患者受益 -> 目前缺少个体化预测/分层工具`

适合的句式：

- `Although ... has improved ..., only a subset of patients ...`
- `However, individualized ... has not been established for ...`

### 2. 命名型方法叙事

- 作者会给核心方法、模块、分型体系起清晰缩写名
- 首次出现时给全称、定义和功能定位
- 后文稳定使用缩写，减少重复表述

应模仿的效果：

- 概念一旦提出，就像论文内部的“品牌名”
- 读者能快速记住模型、模块、分型之间的关系

### 3. 强数字对比，但主要服务于 Results

- 结果段落喜欢用具体数字直接拉开组间差异
- 多组比较时，倾向一次性列出各组关键指标
- 关键统计量紧跟核心结论，而不是藏到句尾角落
- 这种“强数字具体性”默认属于 `Results`，不要机械迁移到 `Discussion`

推荐格式：

- `Subtype A showed the highest ORR (60.0%), compared with subtype B (0%), subtype C (0%), and subtype D (17.6%; p=0.003).`

### 4. 图注像“带路讲解”

- Figure 和 Supplementary Figure 的 legend 都倾向于写成“读图导览”，而不是简短标签
- 每个 panel 不只说图里有什么，还会解释读者应该怎样解码：坐标轴、行列、颜色、点大小、条形图位置、注释轨道、参考线分别代表什么
- 遇到 `NES`、`hallmark`、`C6`、`XCELL` 这类术语时，常顺手补一个简短的人话解释，降低跨领域理解门槛
- 如果图里用了 z-score、标准化、不同 y 轴尺度、隐藏标签等视觉处理，legend 会补一句“为什么这样做”
- 统计符号、删失标记、risk table、显著性星号、灰色单元格等可视符号，作者通常会直接在 legend 中解码

应模仿的效果：

- 读者即使先看图、还没读正文，也能知道图该怎么读
- 非本领域读者能分清“图形元素是什么”和“这些元素在生物学或临床上意味着什么”

### 5. 局限性诚实且成串出现

- Discussion 不回避限制
- 常用 `First, Second, Third...` 组织局限
- 每条局限尽量具体到材料、平台、样本、适用边界
- 局限之后通常顺手给一个未来改进方向

### 6. 叙事主线稳定

- Introduction 的问题、Results 的证据、Discussion 的解释紧密对齐
- Results 以“为什么要做这一分析”为段首驱动
- Discussion 会反过来证明：这些结果为什么能回答一开始的问题

### 7. `Results` 与 `Discussion` 的具体性不是一回事

- `Results` 的具体性：数字、比较、图表锚点、外部验证
- `Discussion` 的具体性：机制解释、临床定位、文献对位、边界条件、未来验证路径
- 不要把“像真实作者的具体性”误做成“所有章节都继续报数”

### 8. `.tex` 源文可定位

- 需要另起一段时才用空行
- 多个证据点仍属于同一段时，优先一行一个“并列证据单元 / panel / 局限项”，方便从 PDF/Word 回跳到 `.tex` 时直接落到对应证据
- Figure legend 尤其要避免把多个 panel 挤成一长行
- 只对自然语言段落使用，不主动重排行敏感 LaTeX 结构

## 分章节写法

### Title

- 明确点出“对象 + 方法/框架 + 结果/用途”
- 不追求华丽，追求可识别性
- 可接受带方法缩写，但不要缩写过载

### Abstract

推荐节奏：

1. 具体问题，而不是宏大背景
2. 提出核心模块/模型，并给短定义
3. 用 1-3 个最关键数字证明结果
4. 用一句话交代临床/科学意义

写作提醒：

- 结果数字尽量选最有区分度的
- 如果有分型体系，第一次就把体系结构说清楚
- 结尾强调“可用于个体化预测/分层/转化”

### Introduction

推荐五段式：

1. 疾病或问题背景
2. 现有方法和已有进展
3. 现有局限，最好编号列出
4. 本研究策略与突破口
5. 本文贡献与产物

关键要求：

- 第 3 段的局限不能泛泛而谈，要有临床或方法学落点
- 第 4 段要把作者的方法命名自然引出
- 第 5 段要说明最终交付了什么，例如模型、分型、工具、验证结果

### Methods

- 强调可复现与工程细节
- 包、软件、版本号、数据源、阈值写清楚
- 小节标题尽量用名词短语
- 如果方法链较长，要保持“数据 -> 特征 -> 模型 -> 验证”的顺序

### Results

每个小节建议遵循：

1. 先说分析目的
2. 再说用什么方法得到证据
3. 然后给出最关键结果和数字
4. 最后放补充验证或跨队列验证

偏好的语气：

- 客观、直接、少修饰
- 让 Figure/Table 成为结果的事实底座
- 多队列验证时，突出“一致性”或“外部验证”
- 同一段里若连续写主结果、对照结果和外部验证，优先一行一个证据点，不插空行

### Figure Legends

- 主标题先点出图的主题对象或核心任务，再进入 panel-by-panel 说明
- 每个 panel 至少交代三层信息：`看什么`、`图形编码分别代表什么`、`需要怎样理解这些编码`
- 优先解释读图必须知道的视觉语法：x/y 轴、行列、颜色方向、点大小、顶部或侧边条形图、注释轨道、聚类方式、参考线、删失标记、risk table
- 如果使用了标准化、z-score、不同 y 轴尺度、隐藏标签或灰色表示“非显著/缺失”，要补一句理由或判读方式
- 缩写首次出现时仍给全称；若术语对非领域读者不直观，可在括号内补一句短定义
- Supplementary Figure 的 legend 也按主图同等详尽度处理，不因为是补充材料就缩成提纲
- 同一条 legend 中，主标题、各 panel、统计或判读补充信息默认按点逐行展开；如果语义仍属同段，则不要插空行
- 不要把整个 Methods 重复一遍；legend 重点是帮助读者读图，而不是重写实验流程

### Discussion

建议结构：

1. 先回到核心问题
2. 解释最重要的发现意味着什么
3. 强调本研究相对已有工作的独特性
4. 系统写局限性
5. 给出可执行的未来方向

关键风格：

- 不做空洞拔高
- 解释结果时要和临床或机制挂钩
- 局限性越具体，越像真实作者
- `First/Second/Third` 这类连续局限若仍属于同一段，优先一条一行
- `Discussion` 不是重新汇报 figure 结果
- 不要连续复述 `AUC`、`ORR`、`HR`、`p value`、图号或统计检验名，除非某一个数字本身就是讨论对象
- 每段默认不超过 `1` 个核心定量锚点；整节默认不追求覆盖所有主图结果
- 如果已经出现 figure 编号、统计检验名和多个效应值的连续堆叠，应优先概括并转入“这意味着什么”
- 每段优先完成以下任务之一：解释主要发现、说明与既有文献的关系、交代临床或生物学转化意义、划清局限性和适用边界

### Conclusion

- 1-3 句就够
- 重申“提出了什么”和“能解决什么”
- 不引入新证据

## 语言特征

### 时态

- Introduction：现在时为主
- Methods：过去时被动语态为主
- Results：过去时
- Discussion：解释用现在时，回顾本工作可混用过去时

### 句长与节奏

- 平均句长偏中等
- 允许用长句承载复杂信息，但不要多层嵌套失控
- 关键判断句通常更短、更直接
- Figure legend 允许略长，但最好保持“一句解释一个视觉单元或一个判读动作”

### 高频连接方式

- 转折：`However`, `Nevertheless`
- 递进：`Moreover`, `Furthermore`, `Notably`, `Intriguingly`
- 因果：`Thus`, `Therefore`

### `.tex` 行级组织

- 新段落用空行
- 同段多点用换行，不用空行
- 这里的“点”优先指并列证据单元、panel 或局限项，不是任意一句话
- 不要为了“看起来整齐”把无关句子重新大面积改行

## 数字与统计表达

- 百分比通常保留 1 位小数
- AUC、相关系数、效应值通常保留 3 位小数
- `p` 值优先精确写，特别小时用 `p < 0.001`
- 多组比较时尽量完整列组别，不要只写“significantly higher”
- 图注里出现数字时，优先让数字紧贴其视觉对象或统计对象，避免孤立堆数值
- 对 `Discussion` 而言，数字只应作为少量解释锚点存在，而不是段落主体

## 缩写与命名

- 首次出现：全称 + 缩写
- “首次出现”按整篇论文所有正文 tex 联合判断，不按当前正在编辑的单个文件判断
- 缩写一旦确定，全文保持一致
- 给方法、模型、分型命名时要可读、可记、可复用

## 可模仿的“作者味”

### 应该模仿

- 用具体问题驱动全文
- 给核心方法一个可记忆的名字
- 在 `Results` 里让关键发现紧跟具体数字
- 局限性写得坦诚且分点清晰
- Discussion 里强调“独特性 + 适用边界 + 为什么重要”

### 不应该模仿成表面套路

- 机械地每段都塞 `However`
- 无根据地硬造缩写
- 为了“像作者”而牺牲清晰度
- 为了增强结论而夸大数字意义
- 在 `Discussion` 中把具体性误写成“继续逐图报数”

## 示例锚点

### 好的开头方向

- “已有治疗对部分患者有效，但缺少个体化预测工具”
- “已有 signature 很多，但转化到临床仍有成本或鲁棒性问题”

### 好的结果表达方向

- “先指出最优亚型，再列其它亚型对比值，再给统计显著性”
- “先给发现，再说明该发现已在独立队列验证”

### `Discussion` 反例与改写方向

- 不佳：在 `Discussion` 连续重报 `AUC`、`ΔAUC`、`ORR`、图号和检验名
- 更佳：把这些结果概括为“独立队列中保留了判别与决策支持信息”，随后解释这为什么支持其作为补充性分层工具

### 好的图注表达方向

- “先说 panel 在展示什么，再解释行列/坐标轴/颜色/点大小各代表什么”
- “遇到专业术语时补一句括号定义，帮助非领域读者不查正文也能读懂”
- “如果图形做了标准化、聚类或坐标缩放，顺手说明这样做的原因”

### 好的局限性方向

- “样本来源导致代表性不足”
- “技术平台限制外推性”
- “人群、地域、性别等泛化能力尚未充分验证”

## 使用提醒

- 当用户要求润色已有论文时，优先保留作者原有叙事主线，只把表达调到更接近这个风格
- 当用户要求从零写作时，先搭出问题主线，再把命名、数字和局限性风格补进去
- 当用户要求写 Figure 或 Supplementary Figure legend 时，默认采用“panel-by-panel 导读 + 视觉编码解码 + 非领域友好解释”的更详尽写法
- 当正文或图注里一个段落承载多个证据点时，默认采用“同段分点换行”，提升 `.tex` 源文可定位性
- 任何数字、机制解释、临床意义都不能为了“像作者”而越过事实边界
