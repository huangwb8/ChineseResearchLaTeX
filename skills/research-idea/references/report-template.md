# Research Idea 报告模板

最终报告只写用户需要看到的内容，不写隐藏工作区路径、agent 内部指令或测试路径。

```markdown
# Research-Idea: {主题或项目名}

## 调研摘要

- 输入资料：{一句话概括}
- 核心方向：{一句话概括}
- 最终建议：优先推进 {最佳问题-假设对编号}

## 多个科学问题-科学假设对

### 候选 1：{短标题}

**科学问题**：{明确研究对象、机制/关系/边界条件}

**可证伪科学假设**：{如果 X 成立，则 Y；如果观察到 Z，则该假设被削弱或推翻}

**关键预测**：
- {预测 1}
- {预测 2}

**反证路径**：
- {什么实验、数据或观察能推翻该假设}

**查新结论**：{Premium 查新后的状态：未研究 / 部分研究但关键缺口存在 / 已充分研究；以及处理决定}

**初步评价**：
- 新颖性：{高/中/低 + 理由}
- 重要性：{高/中/低 + 理由}
- 可行性：{高/中/低 + 理由}

### 候选 2：{短标题}

**科学问题**：{明确研究对象、机制/关系/边界条件}

**可证伪科学假设**：{如果 X 成立，则 Y；如果观察到 Z，则该假设被削弱或推翻}

**关键预测**：
- {预测 1}
- {预测 2}

**反证路径**：
- {什么实验、数据或观察能推翻该假设}

**查新结论**：{Premium 查新后的状态与处理决定}

**初步评价**：
- 新颖性：{高/中/低 + 理由}
- 重要性：{高/中/低 + 理由}
- 可行性：{高/中/低 + 理由}

候选 3-N 必须完整重复上述字段，不得写“同上”。

## 为什么选择这些科学问题-科学假设对

说明这些候选为什么值得进入查新和打磨：
- 它们分别覆盖了哪些机制、争议或知识缺口。
- 它们为什么不是简单重复已有结论。
- 它们如何利用用户提供的资料。

## 查新摘要

| 候选 | 查新结论 | 已有研究覆盖 | 关键缺口 | 处理 |
|------|----------|--------------|----------|------|
| 1 | 未研究 / 部分研究但关键缺口存在 / 已充分研究 | {摘要} | {摘要} | 保留/修改/淘汰 |

只总结和科学问题相关的查新结论，不写完整文献综述。
必须说明查新使用 `research-literature-review` 的 `Premium` 档。

## 最佳科学问题-科学假设对

**最佳选择**：候选 {编号}：{短标题}

**原因**：
- 科学重要性：{理由}
- 新颖性：{理由}
- 可证伪性：{理由}
- 可行性：{理由}
- 解释力：{理由}

**优化后的科学问题**：{最终版本}

**优化后的可证伪科学假设**：{最终版本}

**最关键的反证实验/观察**：{一到三条}

## 风险与下一步

- 最大不确定性：{说明}
- 最容易失败的地方：{说明}
- 下一步建议：{不是完整实验计划，只写验证该假设的最小下一步}
```
