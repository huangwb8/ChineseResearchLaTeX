# 查新判定指南

目标不是写综述，而是判断某个“科学问题-假设”是否仍有研究价值。

## 查新输入

把每个候选先交给 `research-topic-extractor`，得到：
- topic：一句话主题。
- keywords：5-10 个英文检索术语。
- core_questions：2-5 个查新问题。

再调用 `research-literature-review`，档位必须为 `Premium`。
输出目录必须限定到隐藏工作区：`.research-idea/run-*/novelty/Cx/`。

## 判定分级

### 未研究

满足以下多数条件：
- 没有发现直接提出相同科学问题的研究。
- 没有发现检验同一机制假设的研究。
- 相关研究只覆盖背景、相关现象或相邻技术。

处理：保留，进入多轮独立审查。

### 部分研究但关键缺口存在

满足以下任一条件：
- 已有研究关注相同对象，但没有检验该假设的关键预测。
- 已有研究检验相似假设，但研究对象、条件、尺度或机制不同。
- 已有结论冲突，且候选假设能提出可反证解释。

处理：保留但必须改写，让科学问题明确落在缺口上。

### 已充分研究

满足以下任一条件：
- 已有高质量研究直接回答该科学问题。
- 该假设或等价假设已被系统检验。
- 该候选只是在换词重复已有结论。

处理：淘汰或大幅重构，再提出下一组候选。

## 证据记录

中间证据写入隐藏工作区，最终报告只保留简要查新摘要：
- 已有研究覆盖了什么。
- 仍缺什么。
- 为什么保留、修改或淘汰该候选。

每个候选必须写出 `novelty-decision.json`：

```json
{
  "candidate_id": "C1",
  "novelty_status": "未研究 / 部分研究但关键缺口存在 / 已充分研究",
  "direct_answer": "已有研究是否直接回答该科学问题",
  "equivalent_hypothesis_tested": "是否已有等价假设被检验",
  "key_gap": "关键缺口",
  "decision": "保留 / 修改 / 淘汰"
}
```
