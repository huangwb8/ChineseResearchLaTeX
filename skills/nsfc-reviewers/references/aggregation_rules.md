# 跨组聚合规则

此文件定义如何聚合多个评审组（多个 thread）的 `panel_review.md` 输出。

## 聚合流程

1. 读取所有组报告：从 N 个 thread 读取 `panel_review.md`
2. 跨组共识识别：
   - 至少 `min_panels` 个组指出同一问题 → 跨组共识
   - `min_panels = ceil(N * consensus_threshold)`（默认 `consensus_threshold = 0.6`）
3. 严重度二次升级：
   - 跨组共识问题：P2→P1→P0（P0 不再升级）
4. 去重合并：
   - 相同问题的不同表述合并为一条
   - 保留最丰富的证据锚点与最详细的建议
5. 来源标注：
   - 跨组共识：标注“跨组共识（N 组）”
   - 单一组意见：标注“来自组 GXXX”
6. 阶段判断聚合：
   - 对函评/会评分别统计各组“给过/不给过”结论
   - 若最终“函评不给过”，则“会评”必须同步输出“不给过”
   - 会评判断要比函评更严格，可因相对竞争力不足而单独判“不给过”
7. 优先级排序：
   - P0（跨组共识）→ P0（单组）→ P1（跨组共识）→ P1（单组）→ P2

## 输出结构（markdown）

```markdown
# 国自然标书评审意见（N 组独立专家，每组 7 位专家）

## 评审配置
- 评审组数：{panel_count} 组
- 每组专家：7 位
- 总专家人次：{panel_count * 7} 人次

## 阶段判断（基于当前版本直接送审）

### 函评
- 结论：给过 / 不给过
- 判断把握：高 / 中 / 低
- 评审组倾向：X/N 组判给过，Y/N 组判不给过
- 主要依据：...
- 翻盘关键（仅在不给过时必写）：...

### 会评
- 结论：给过 / 不给过
- 判断把握：高 / 中 / 低
- 评审组倾向：X/N 组判给过，Y/N 组判不给过
- 主要依据：...
- 翻盘关键（仅在不给过时必写）：...

## 跨组共识（多组一致指出）

### P0 级（致命问题，必须修改）

### P1 级（重要问题，建议修改）

### P2 级（可选改进）

## 独立观点（单一组提出）

### 来自组 G001

## 修改建议汇总（最小可行修改序列）

## 附录：各组原始评审报告
```
