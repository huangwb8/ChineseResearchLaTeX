# `项目指南.md` 结构映射（用于"融入式更新"）

目标：将新增洞见写入既有结构中，不新增 `##` 级标题、不改动既有标题文本/顺序/编号。

## 放置规则（从高到低优先级）

1) **能减少全文口径漂移的内容**（术语、缩写、核心概念定义、大小写、对照口径）优先写入 `## 命名与术语`（或其相邻的"核心概念/评价指标"小节）。
2) **能减少评审质疑的内容**（差异化要点、方法选择理由、关键设计决策）优先写入 `## 文献综述与创新性` 或 `## 评审专家可能的问题及回答`。
3) **能让技术路线更可检验的内容**（评估指标、验证拆分、对照/消融设计）优先写入 `## 基本原理` 或 `## 落点清单` 的对应条目。
4) **风险与对策**统一写入 `## 研究风险`。
5) **可核验事实锚点**（源码位置、参数、流程）统一写入 `## 附录`。

## 典型洞见 → 推荐落点

- "新增一条写作禁区/句式建议/引用策略" → `## 写作哲学`
- "统一某个英文缩写/大小写/连字符/术语对齐" → `## 命名与术语`
- "补充流程拆分/核心概念定义/评估标准/指标口径" → `## 命名与术语` 或 `## 基本原理`
- "补充评估指标/校准方法/验证策略的写法" → `## 基本原理` + `## 落点清单`
- "需要在文档哪一节强调什么" → `## 落点清单`
- "新增对照/消融" → `## 文献综述与创新性` 或 `## 落点清单`
- "新增评审问答口径" → `## 评审专家可能的问题及回答`
