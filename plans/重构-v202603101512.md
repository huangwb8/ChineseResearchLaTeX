# NSFC LaTeX 模板重构计划

**版本**: v202603101512-optimized  
**创建时间**: 2026-03-10 15:12  
**更新时间**: 2026-03-10 15:30  
**目标**: 将 projects 目录下的共性配置抽取为 `nsfc-common` LaTeX 包，采用**分层设计**保证灵活性，并提供 Python 脚本管理包的安装和更新

---

## 一、项目背景

### 当前问题

1. **代码重复**: 三个项目（General/Local/Young）的 `@config.tex` 约 90% 内容相同
2. **维护成本高**: 修改配置需要同步更新三个文件
3. **一致性风险**: 手动同步容易遗漏，导致项目间不一致

### 重构目标

1. **DRY 原则**: 消除重复配置，单一真相来源
2. **易维护**: 修改一处，所有项目受益
3. **保持灵活性**: 用户可以选择不同的使用模式，从完全自定义到开箱即用 ⭐
4. **用户友好**: 提供类似 `pip install` 的安装体验
5. **向后兼容**: 不破坏现有用户的使用习惯，支持渐进式迁移

---

## 二、技术方案

### 2.1 设计哲学：分层架构 ⭐

采用**三层配置架构**，确保在提升可维护性的同时保持用户的自定义灵活性：

\`\`\`
┌─────────────────────────────────────────────────────┐
│  第 3 层：用户自定义层 (@config.tex)                 │
│  - 优先级：最高                                      │
│  - 用途：覆盖包的默认配置，实现深度定制              │
│  - 示例：更换宏包、调整命令、自定义样式              │
└─────────────────────────────────────────────────────┘
                        ↓ 覆盖
┌─────────────────────────────────────────────────────┐
│  第 2 层：项目类型层 (nsfc-common 包选项)            │
│  - 优先级：中等                                      │
│  - 用途：根据项目类型（general/local/young）         │
│         应用差异化配置（页边距、特殊命令等）         │
└─────────────────────────────────────────────────────┘
                        ↓ 继承
┌─────────────────────────────────────────────────────┐
│  第 1 层：基础配置层 (nsfc-common 核心)              │
│  - 优先级：最低                                      │
│  - 用途：提供 90% 的共性配置（宏包、字体、行距等）   │
│  - 特点：稳定、经过验证、适合大多数用户              │
└─────────────────────────────────────────────────────┘
\`\`\`

### 2.2 使用模式 ⭐

用户可以根据需求选择不同的使用模式：

| 模式 | 适用人群 | 灵活性 | 维护成本 | 使用方式 |
|------|---------|--------|---------|---------|
| **完全包模式** | LaTeX 新手、追求稳定 | ★☆☆ | ★★★ | \`\\usepackage[type=general]{nsfc-common}\` |
| **最小包模式** | 高级用户、深度定制 | ★★★ | ★☆☆ | \`\\usepackage[type=general,minimal]{nsfc-common}\` + 自定义 |
| **混合模式** | 大多数用户（推荐） | ★★☆ | ★★☆ | 包 + @config-override.tex 覆盖 |
| **独立模式** | 完全自主控制 | ★★★ | ★☆☆ | 不使用包，保留完整 @config.tex |

### 2.4 包设计

#### nsfc-common.sty 结构（增强版）⭐

```latex
\NeedsTeXFormat{LaTeX2e}
\ProvidesPackage{nsfc-common}[2026/03/10 v1.0.0 NSFC Common Package]

% ========== 选项定义 ==========
% 项目类型选项
\newif\ifNSFC@general
\newif\ifNSFC@local
\newif\ifNSFC@young

\DeclareOption{general}{\NSFC@generaltrue}
\DeclareOption{local}{\NSFC@localtrue}
\DeclareOption{young}{\NSFC@youngtrue}

% 灵活性选项（新增）⭐
\newif\ifNSFC@minimal      % 最小化模式
\newif\ifNSFC@nopackages   % 不加载宏包（用户自己加载）
\newif\ifNSFC@nofonts      % 不配置字体（用户自己配置）
\newif\ifNSFC@notitles     % 不配置标题格式（用户自己配置）

\DeclareOption{minimal}{\NSFC@minimaltrue}
\DeclareOption{nopackages}{\NSFC@nopackagestrue}
\DeclareOption{nofonts}{\NSFC@nofontstrue}
\DeclareOption{notitles}{\NSFC@notitlestrue}

\ProcessOptions\relax

% ========== 用户覆盖钩子（新增）⭐ ==========
% 在关键位置提供钩子，允许用户插入自定义代码
\providecommand{\NSFCBeforePackages}{}  % 宏包加载前钩子
\providecommand{\NSFCAfterPackages}{}   % 宏包加载后钩子
\providecommand{\NSFCBeforeFonts}{}     % 字体配置前钩子
\providecommand{\NSFCAfterFonts}{}      % 字体配置后钩子
\providecommand{\NSFCUserOverride}{}    % 用户最终覆盖钩子

% 执行用户前置钩子
\NSFCBeforePackages

% ========== 共性配置 ==========
% 1. 宏包引用（90% 相同）
\ifNSFC@nopackages
  % 用户选择不加载宏包，跳过
\else
  \usepackage[english]{babel}
  \usepackage{xcolor}
  \usepackage{graphicx}
  \usepackage{amsmath}
  \usepackage{amssymb}
  % ... 其他 20+ 个包
\fi

% 执行用户中间钩子
\NSFCAfterPackages

% 2. 字号定义（100% 相同，始终加载）
\newcommand{\chuhao}{\fontsize{42pt}{\baselineskip}\selectfont}
\newcommand{\xiaochuhao}{\fontsize{36pt}{\baselineskip}\selectfont}
\newcommand{\yihao}{\fontsize{28pt}{\baselineskip}\selectfont}
\newcommand{\erhao}{\fontsize{21pt}{\baselineskip}\selectfont}
\newcommand{\xiaoerhao}{\fontsize{18pt}{\baselineskip}\selectfont}
\newcommand{\sanhao}{\fontsize{15.75pt}{\baselineskip}\selectfont}
\newcommand{\sihao}{\fontsize{14pt}{\baselineskip}\selectfont}
\newcommand{\xiaosihao}{\fontsize{12pt}{\baselineskip}\selectfont}
\newcommand{\wuhao}{\fontsize{10.5pt}{\baselineskip}\selectfont}
\newcommand{\xiaowuhao}{\fontsize{9pt}{\baselineskip}\selectfont}
\newcommand{\liuhao}{\fontsize{7.875pt}{\baselineskip}\selectfont}
\newcommand{\qihao}{\fontsize{5.25pt}{\baselineskip}\selectfont}

% 3. 字体配置（100% 相同）
\ifNSFC@nofonts
  % 用户选择不配置字体，跳过
\else
  \NSFCBeforeFonts
  \ifwindows
    % Windows 字体
    \setCJKmainfont{SimSun}
    \setCJKsansfont{SimHei}
    \setCJKmonofont{FangSong}
  \else
    % macOS/Linux 字体
    \setCJKmainfont{Songti SC}
    \setCJKsansfont{Heiti SC}
    \setCJKmonofont{STFangsong}
  \fi
  \NSFCAfterFonts
\fi

% 4. 行距与段落（100% 相同）
\renewcommand{\baselinestretch}{1.0}
\setlength{\parindent}{2em}
\setlength{\parskip}{0pt}

% 5. 标题格式（100% 相同）
\ifNSFC@notitles
  % 用户选择不配置标题格式，跳过
\else
  \titleformat{\section}{\sanhao\bfseries}{\thesection}{1em}{}
  \titleformat{\subsection}{\sihao\bfseries}{\thesubsection}{1em}{}
  \titleformat{\subsubsection}{\xiaosihao\bfseries}{\thesubsubsection}{1em}{}
\fi

% 6. 参考文献（100% 相同）
\renewenvironment{thebibliography}[1]{%
  \section*{\refname}%
  \list{\@biblabel{\@arabic\c@enumiv}}%
       {\settowidth\labelwidth{\@biblabel{#1}}%
        \leftmargin\labelwidth
        \advance\leftmargin\labelsep
        \@openbib@code
        \usecounter{enumiv}%
        \let\p@enumiv\@empty
        \renewcommand\theenumiv{\@arabic\c@enumiv}}%
  \sloppy
  \clubpenalty4000
  \@clubpenalty \clubpenalty
  \widowpenalty4000%
  \sfcode`\.\@m}
 {\def\@noitemerr
   {\@latex@warning{Empty `thebibliography' environment}}%
  \endlist}

% ========== 差异化配置 ==========
% 根据项目类型设置不同的页边距
\ifNSFC@general
  \geometry{left=3.00cm,right=2.75cm,top=2.63cm,bottom=2.96cm}
  \usepackage{wasysym}
  \newcommand{\ssssubtitle}[1]{\textcircled{\raisebox{-0.9pt}{#1}}}
\fi

\ifNSFC@local
  \geometry{left=3.20cm,right=3.20cm,top=2.54cm,bottom=2.54cm}
  \usepackage{pifont}
  \newcommand{\ssssubtitle}[1]{\ding{172}}
\fi

\ifNSFC@young
  \geometry{left=3.20cm,right=2.94cm,top=2.67cm,bottom=2.91cm}
  \usepackage{wasysym}
  \newcommand{\ssssubtitle}[1]{\ding{172}}
  \raggedbottom
\fi

% ========== 用户最终覆盖钩子 ⭐ ==========
% 在包加载的最后，执行用户的覆盖配置
\NSFCUserOverride
```

#### 使用示例 ⭐

**模式 1：完全包模式（推荐新手）**

```latex
% main.tex
\documentclass{article}
\usepackage[type=general]{nsfc-common}  % 一行搞定

\begin{document}
% 你的内容
\end{document}
```

**模式 2：最小包模式（推荐高级用户）**

```latex
% main.tex
\documentclass{article}

% 只加载字号定义和差异化配置，其他自己控制
\usepackage[type=general,nopackages,nofonts,notitles]{nsfc-common}

% 自己加载需要的宏包
\usepackage{xcolor}
\usepackage{graphicx}
% ...

% 自己配置字体
\setCJKmainfont{Noto Serif CJK SC}

% 自己配置标题格式
\titleformat{\section}{...}

\begin{document}
% 你的内容
\end{document}
```

**模式 3：混合模式（推荐大多数用户）⭐**

```latex
% main.tex
\documentclass{article}

% 定义覆盖钩子
\renewcommand{\NSFCAfterPackages}{%
  % 在包加载后，添加自己需要的包
  \usepackage{tikz}
  \usepackage{pgfplots}
}

\renewcommand{\NSFCUserOverride}{%
  % 覆盖包的默认配置
  \geometry{left=2.5cm,right=2.5cm}  % 自定义页边距
  \renewcommand{\baselinestretch}{1.2}  % 自定义行距
}

\usepackage[type=general]{nsfc-common}

% 或者在包加载后覆盖
\input{extraTex/@config-override.tex}

\begin{document}
% 你的内容
\end{document}
```

**模式 4：独立模式（完全自主控制）**

```latex
% main.tex
\documentclass{article}

% 不使用包，使用完整的 @config.tex
\input{extraTex/@config.tex}

\begin{document}
% 你的内容
\end{document}
```

### 2.5 Python 包管理器

#### 核心功能

1. **安装**: `python install.py install`
   - 检测 TEXMFHOME 路径
   - 复制 .sty 文件到 `$TEXMFHOME/tex/latex/nsfc-common/`
   - 保存版本信息

2. **升级**: `python install.py upgrade`
   - 检查当前版本 vs 已安装版本
   - 备份旧版本
   - 安装新版本

3. **卸载**: `python install.py uninstall`
   - 删除安装的包文件
   - 清理版本信息

4. **状态查询**: `python install.py status`
   - 显示当前版本、已安装版本
   - 显示安装路径

#### 技术细节

- **跨平台支持**: 自动检测 Windows/macOS/Linux
- **版本管理**: 在安装目录保存 `.version` 文件
- **备份机制**: 升级前自动备份旧版本到 `.backup` 目录
- **错误处理**: 友好的错误提示和回滚机制

## 四、风险评估与应对

### 4.1 技术风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|------------|
| LaTeX 包选项机制不熟悉 | 高 | 中 | 参考 ctex、geometry 等成熟包的实现 |

## 附录 D：设计决策说明 ⭐

### D.1 为什么选择分层设计？

**问题**：用户担心重构后失去自定义灵活性。

**解决方案**：采用三层配置架构，确保用户始终可以覆盖任何默认配置。

**优势**：
1. **新手友好**：可以直接使用包，无需关心细节
2. **高手友好**：可以通过钩子和选项实现深度定制
3. **渐进式**：用户可以从简单模式逐步过渡到复杂模式

### D.2 为什么提供四种使用模式？

**问题**：不同用户有不同的需求和技术水平。

**解决方案**：提供四种模式，覆盖从新手到高手的所有场景。

| 用户类型 | 推荐模式 | 理由 |
|---------|---------|------|
| LaTeX 新手 | 完全包模式 | 开箱即用，无需配置 |
| 有经验用户 | 混合模式 | 平衡灵活性与维护性 |
| LaTeX 高手 | 最小包模式 | 完全自主控制 |
| 保守用户 | 独立模式 | 保持原有工作流 |

### D.3 为什么保留独立模式？

**问题**：部分用户可能完全不想使用包。

**解决方案**：保留完整的 `@config.tex`，用户可以选择不使用包。

**优势**：
1. **向后兼容**：旧用户无需修改任何代码
2. **降级方案**：如果包出现问题，可以立即切换回独立模式
3. **学习参考**：新用户可以对比包和独立配置，理解包的工作原理

### D.4 钩子机制的设计考虑

**钩子位置选择**：
- `\NSFCBeforePackages`：在加载宏包前，用户可以设置环境变量或加载前置包
- `\NSFCAfterPackages`：在加载宏包后，用户可以加载依赖于默认包的额外包
- `\NSFCBeforeFonts`：在配置字体前，用户可以设置字体路径
- `\NSFCAfterFonts`：在配置字体后，用户可以微调字体设置
- `\NSFCUserOverride`：在包加载的最后，用户可以覆盖任何默认配置

**设计原则**：
1. **最小惊讶**：钩子名称清晰，位置符合直觉
2. **充分灵活**：覆盖包加载的所有关键阶段
3. **易于使用**：提供 `\providecommand`，用户不定义也不会报错
