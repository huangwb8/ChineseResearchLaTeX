## 1. 背景与问题定义

你当前维护的旧 Skill 集合（如 `nsfc-rationale-writer`、`nsfc-aims-writer`、`nsfc-foundation-conditions-writer` 等）总体是“按章节独立写作”的思路，这是对的；但模板换版后主要出现了两类不适配：

1) **板块边界变化**：新模板（以 `projects/NSFC_Young/main.tex` 与 `projects/NSFC_Young/template/2026年最新word模板-青年科学基金项目（C类）-正文.docx` 为例）把板块压缩为更“自由提纲”的形态：  
   - `（一）立项依据`：只给一句括号说明（价值/必要性），不再提供细分小标题  
   - `（二）研究内容`：强调“按研究自身逻辑”，并显式要求提炼**特色与创新点**与**年度研究计划**  
   - `（三）研究基础`：明确包含“研究基础与可行性分析 + 研究风险应对”，并新增/强化在研项目与结题项目说明  

2) **旧 Skill 的“隐含假设”太强**：旧技能文档中存在较多项目特化口径（例如强依赖 `项目指南.md`、特定领域术语与内部方法体系等），这会导致新模板下复用困难（换课题就不适用）。

目标不是“把旧 Skill 全部推倒重写”，而是把它们的**有效能力**（结构化写作、可检验目标、对照与风险、证据链、字数控制、引用守护）迁移到“以新模板为契约”的新 Skill 体系里。

以 `projects/NSFC_Young/main.tex` 的 `extraTex/*.tex` 为“落笔位置”示例：

## 3. Skill 体系重组建议（按“3 个主 Skill + 若干子 Skill”）

### 3.1 三个主 Skill（你提到最重要的三个）

建议把“用户心智模型”对齐到新模板的 3 个主板块，形成 3 个主 Skill（可作为入口/编排器），它们只负责：

- 明确输入信息表（见第 4 节）
- 选择/编排子 Skill
- 做跨章节一致性校验（调用 writing-core）

推荐的主 Skill（命名可调整，但建议直指板块）：

1) `nsfc-justification-writer`（对应 `（一）立项依据`）  
2) `nsfc-research-content-writer`（对应 `（二）研究内容`，并编排创新点与年度计划）  
3) `nsfc-research-foundation-writer`（对应 `（三）研究基础`，并编排工作条件与风险应对）

### 3.2 子 Skill（迁移旧能力，保持单一职责）

把旧 Skill 中“成熟且可复用”的能力拆到子 Skill，主 Skill 只做 orchestration：

- 从 `nsfc-rationale-writer` 迁移：立项依据的论证结构（价值→不足→断点→切入点→小结），但去掉特定课题口径  
- 从 `nsfc-aims-writer` 迁移：研究内容/目标/关键科学问题的“三件套”，但输出对齐到 `2.1`（并与创新/计划解耦）  
- 从 `nsfc-innovation-writer` 迁移：创新点“对照坐标系”写法（相对创新，不绝对化）→ 输出 `2.2`  
- 从 `nsfc-workplan-writer` 迁移：三年计划“整年 + 里程碑 + 交付物”→ 输出 `2.3`  
- 从 `nsfc-foundation-conditions-writer` 迁移：研究基础“证据链”+工作条件“资源对位”→ 输出 `3.1/3.2`  
- 从 `nsfc-methods-feasibility-writer` 迁移：可行性“四维度”与对照/消融/泄漏防控等写法，但要把它放到新模板的 `3.1`（因为模板明确写了“可行性分析/风险应对”）  
- `nsfc-bib-manager` 继续作为强制依赖：新增引用必须核验入库  
- `nsfc-writing-core` 升级为“守门员”：模板不破坏 + 全稿一致性 + 风险质疑预防

## 4. 去特化：统一输入信息表（让 Skill 可迁移到任何课题）

旧 Skill 的最大复用障碍是“默认你在写某个具体课题”。建议在系统新 Skill 中引入一份统一的“信息表”（可 JSON/YAML/Markdown），作为所有写作 Skill 的唯一上游依赖。

最小信息表（MVP，能支撑三大板块）建议字段：

- **研究对象/场景**：疾病/材料/工程场景、目标人群/体系边界  
- **临床/工程痛点**：一句话问题定义 + 现有方案瓶颈  
- **核心假说/科学问题**：1 句假说 + 1–3 条关键科学问题（断点式）  
- **总体目标与子目标**：3–4 个可检验子目标（每个都有指标/对照/数据来源）  
- **技术路线概览**：输入→处理→模型/实验→验证→交付  
- **前期基础**：已完成成果（可核验）+ 预实验现象（可描述）  
- **资源与条件**：数据/样本/算力/平台/合作与合规  
- **风险清单**：技术/进度/资源风险各 1–3 条 + 预案  

写作时：  
`（一）` 主要消费“痛点/价值/不足/假说”；`（二）` 主要消费“目标/路线/验证”；`（三）` 主要消费“前期基础/资源/风险”。

## 5. 可选但高收益的“新模板增量 Skill”

虽然你当前优先 3 大板块，但新模板里有几个“强制且易漏”的点，建议用小 Skill 快速兜底：

1) `nsfc-risk-response-writer`（输出到 `extraTex/3.1.研究基础.tex` 的“风险应对”段落模板）  
2) `nsfc-projects-status-writer`（输出 `extraTex/3.3.*` 与 `extraTex/3.4.*` 的表格/条目；强格式、低创造性，适合模板化）  
3) `nsfc-ai-usage-disclosure-writer`（输出 `extraTex/4.5.生成式人工智能.tex`：是否使用、使用位置、如何核验、如何避免不当使用；这是 2026 模板的硬要求之一）

这些小 Skill 不需要“写得漂亮”，只需要“合规、完整、不踩雷”。

## 6. 迁移实施路线（建议按 2 个 Sprint 做完）

### Sprint A（先可用，再完美）：对齐新模板契约

1) **抽取旧 Skill 的“可复用能力”**：结构、检查清单、风险/对照写法、字数控制策略  
2) **建立 2026 模板 profile**（推荐 YAML）：文件落点、板块边界、必填项、禁止改动项  
3) **实现 3 个主 Skill 的最小闭环**：能在新模板对应文件里产出合格内容  
4) **writing-core 加守门校验**：  
   - 不改 `main.tex`/提纲标题/括号说明  
   - 只写 `extraTex/*.tex`  
   - 全稿术语一致（从信息表出发）

### Sprint B（质量提升）：一致性与“强制项兜底”

1) 引入 `nsfc-risk-response-writer` 与 `nsfc-ai-usage-disclosure-writer`  
2) 引入“跨章节一致性检查”（最少做：术语、目标-方法-验证闭环、风险对位）  
3) 为每个主 Skill 增加 1 个最小示例与自检清单（不追求大而全）

## 7. 验收标准（建议写进每个 Skill 的 Definition of Done）

**功能验收（必须）**：

- 输出文件位置正确（与新模板 `extraTex/*.tex` 一一对应）
- 不修改提纲（`main.tex` 或等价主文件的标题/括号说明不被破坏）
- `（一）` 能讲清价值与切入点；`（二）` 有清晰的目标-路径-验证；`（三）` 有证据链 + 条件对位 + 风险预案

**质量验收（建议）**：

- 术语与缩写首现定义，全稿一致
- 每个子目标至少有：指标 + 对照/基线 + 验证数据来源
- 无“无法核验的事实/引用”（引用必须经 `nsfc-bib-manager`）
